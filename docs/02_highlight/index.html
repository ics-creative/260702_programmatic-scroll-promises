概要 / Overview

Programmatic Scroll Promises は、JavaScriptから開始したスクロールの完了を Promiseとして待てるようにする提案です。スムーススクロールのあとにUIを更新したい場合、 スクロール時間を推測するのではなく、完了を待ってから次の処理へ進めます。

Programmatic Scroll Promises let JavaScript wait for a programmatic scroll to finish. When an interface needs to update after smooth scrolling, the code can wait for completion instead of guessing a timeout.

対応メソッド / Available methods

主な対象は、window.scrollTo()、window.scrollBy()、 Element.scrollTo()、Element.scrollBy()、Element.scrollIntoView() など、スクリプトからスクロールを開始するメソッドです。behavior: "smooth" を指定した場合も、 スクロールが終わってから後続処理を実行できます。

The proposal applies to methods that start scrolling from script, such as window.scrollTo(), window.scrollBy(), Element.scrollTo(), Element.scrollBy(), and Element.scrollIntoView(). With behavior: "smooth", follow-up work can run after the scroll reaches its destination.

対応ブラウザ / Browser support

このデモは、Programmatic Scroll Promises を試せるブラウザで動作を確認できます。対応している環境では await window.scrollTo() の後にハイライト処理が実行され、 未対応の環境ではスクロール完了を厳密には待てません。

This demo is intended for browsers that can try Programmatic Scroll Promises. In supported environments, the highlight starts after await window.scrollTo() resolves. In unsupported environments, the browser cannot strictly wait for the scroll completion.

動機 / Motivation

これまでは、スクロール後に何かを見せたいときに setTimeout() や scroll イベントの監視で完了タイミングを推測する必要がありました。 Programmatic Scroll Promises があれば、スクロール完了を処理の境界として扱えます。

Previously, code often used setTimeout() or listened to scroll events to guess when a scroll animation had ended. Programmatic Scroll Promises make scroll completion a clear point in the control flow.
